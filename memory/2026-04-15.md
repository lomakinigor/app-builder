- Единственный баг: `raw.split('\n')` кидает TypeError если `raw = null | undefined` → фикс: `const safeRaw = typeof raw === 'string' ? raw : ''`
- Keyword fallback может агрессивно матчить: текст "Teams struggle..." попадает под targetUsers keyword "team" — тест скорректирован под реальное поведение
- Все scalar fields всегда non-empty string (fallback chain зашита)
- Все array fields всегда массивы (risks/opportunities/openQuestions могут быть пустыми, targetUsers — минимум 1 элемент)
- `briefSource` всегда `'imported'`, `sourceIds` всегда `[artifactId]`

### Покрытые сценарии (7 групп)
- A. Happy path: full structured markdown → все секции, count ≥ 4, warnings = []
- B. Empty/null/undefined: no throw, safe shape, extractedSectionCount = 0, 2 warnings
- C. Missing sections: fallback strings на scalar, [] на arrays (кроме targetUsers)
- D. Malformed: headers без body, дубликаты (last wins), random text
- E. ideaDraft fallback: заполняет problem/targetUser когда секций нет; labeled > ideaDraft
- F. Shape invariants: it.each по 7 кейсам — arrays stay arrays, strings stay strings, fixed fields
